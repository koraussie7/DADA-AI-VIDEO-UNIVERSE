Help

ThunderPay is an Eltoo channel peer-to-peer payment application.

You can create a channel with any of your Maxima contacts

Payments occur on Layer 2, off-chain, and not on the main base chain.

Setting up and closing a payment channel both require an on-chain transaction

Every 'Send' during that is performed off-chain

You can only send and receive as much as is in the channel in total.

And.. this system works with both Minima as well as Tokens!

As we test this new technology.. the current maximum channel size is set to 1000 Minima / Tokens per User.

Technical

Starting a channel involves creating the initial funding, trigger and settlement transactions.

This involves sending multiple Maxima messages back and forward where the transactions are ONLY signed by you when you have the correct signed response transactions.

Once you have all the correct signed transactions, the Funding transaction is posted on-chain.

Sending money involves creating a new signed Update and Settlement transaction.. but NOT posting them on-chain. These are Layer 2 transactions.

These transactions include a sequence number.. which means the LATEST transactions can always be posted and earlier transactions can be overridden.

You can keep doing this as long as you like. Only the Users in the channel are involved.

When you are ready to close the channel there are 2 options :

1) Close the channel cooperatively - which only requires 1 transaction and occurs immediately.

2) Close the channel un-cooperatively (if you can't get hold of the User ) - which requires sending 3 transactions in the correct order.. the trigger, latest update and latest settlement.

The fact that you can close the channel unilaterally allows you to ensure that the CORRECT final amounts are always made to each user.

Since these transactions are not occuring on-chain.. all users can perform payments in this way, at the same time.. allowing for almost limitless transactions per second globally.
